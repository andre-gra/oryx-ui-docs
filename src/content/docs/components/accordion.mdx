# Accordion

The `Accordion` component allows users to show and hide sections of related content on a page. It's built on Radix UI primitives for accessibility.

## Import

```tsx
import { Accordion } from '@andre-gra/oryx-ui';
```

## Usage

```tsx
import { Accordion } from '@andre-gra/oryx-ui';

<Accordion
  items={[
    { mainText: "Section 1", collapsibleText: "Content 1" },
    { mainText: "Section 2", collapsibleText: "Content 2" }
  ]}
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `AccordionItem[]` | Required | Array of accordion items to display |

### AccordionItem

| Property | Type | Description |
|----------|------|-------------|
| `mainText` | `string` | The header text displayed on the accordion trigger |
| `collapsibleText` | `string` | The content revealed when the accordion item is expanded |

## Examples

### Basic Accordion

```tsx
<Accordion
  items={[
    { mainText: "Is it accessible?", collapsibleText: "Yes. It adheres to the WAI-ARIA design pattern." },

    { mainText: "Is it animated?", collapsibleText: "Yes. It uses CSS animations for smooth transitions." }
  ]}

```

## Accessibility

The Accordion component is built with accessibility in mind:

- Full keyboard navigation support
- Proper ARIA attributes for screen readers
- Focus management between items
